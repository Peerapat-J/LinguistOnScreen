# ScreenTranslate v1.1 개선 기획서

> 작성일: 2026-03-02
> 기반: v1.0 (Phase 1~5 완료) 실사용 피드백

---

## 배경

v1.0 실사용 테스트에서 다음 개선 필요사항이 확인됨:
- 번역 결과를 다시 확인할 방법이 없음
- 언어팩 미설치 시 사용자가 상태를 알 수 없음
- 설정창/팝업 크기가 작아 내용이 잘림
- 원문과 번역문을 함께 비교할 수 없음
- 연속 번역 시 이전 요청이 에러로 표시됨

---

## 개선 항목

### 1. 번역 히스토리

**목적**: 이전 번역 결과를 다시 확인하고 활용할 수 있게 한다.

**요구사항**:
- 번역 완료/실패 기록을 시간순으로 저장
- 각 기록에 포함할 정보:
  - 타임스탬프
  - 원문 텍스트
  - 번역된 텍스트 (성공 시)
  - 실패 사유 (실패 시)
  - 소스 언어 → 타겟 언어
- 메뉴바에서 "히스토리" 항목으로 접근
- 히스토리 항목 클릭 시 번역문 복사
- 히스토리 전체 삭제 기능

**저장 방식**:
- v1.1: 메모리 내 배열 (앱 실행 중에만 유지, 최대 50건)
- 향후: SwiftData로 영구 저장 확장 가능

**UI 설계**:
```
┌─────────────────────────────────────┐
│  번역 히스토리                    ✕  │
├─────────────────────────────────────┤
│  16:35  영어 → 한국어               │
│  "Registered Projects" →            │
│  "등록된 프로젝트"            [복사] │
├─────────────────────────────────────┤
│  16:32  영어 → 한국어               │
│  "Active Project:" →                │
│  "활성 프로젝트:"             [복사] │
├─────────────────────────────────────┤
│  16:30  영어 → 한국어        ⚠ 실패 │
│  "Some text" →                      │
│  언어팩 미설치                       │
├─────────────────────────────────────┤
│               [히스토리 삭제]        │
└─────────────────────────────────────┘
```

**데이터 모델**:
```swift
struct TranslationRecord {
    let id: UUID
    let timestamp: Date
    let sourceText: String
    let translatedText: String?     // nil이면 실패
    let failureReason: String?      // nil이면 성공
    let sourceLanguage: String      // "auto" 또는 언어 코드
    let targetLanguage: String
}
```

**구현 위치**:
- `TranslationHistory` 클래스 (Core/)
- `HistoryView` (UI/History/)
- `HistoryWindow` (UI/History/)
- `MenuBarView`에 "히스토리" 메뉴 항목 추가

### 2. 언어팩 상태 표시 및 다운로드

**목적**: 사용자가 언어팩 설치 상태를 한눈에 파악하고, 미설치 언어를 쉽게 다운로드할 수 있게 한다.

**요구사항**:
- 설정의 언어 피커에서 각 언어의 설치 상태 표시
  - ✅ 다운로드됨 (즉시 사용 가능)
  - ⚠️ 미다운로드 (다운로드 필요)
  - ⏳ 다운로드 중
- 미설치 언어를 선택하면 다운로드 확인 다이얼로그 표시
- 다운로드 진행률 표시

**Apple API**:
```swift
import Translation

// 언어 쌍의 지원 상태 확인
let availability = LanguageAvailability()
let status = await availability.status(
    from: Locale.Language(identifier: "en"),
    to: Locale.Language(identifier: "ko")
)
// status: .installed, .supported (다운로드 필요), .unsupported
```

**UI 설계**:
```
┌─────────────────────────────────────┐
│ 원문 언어                            │
│ ┌─────────────────────────────────┐ │
│ │ 자동 감지                    ▾  │ │
│ └─────────────────────────────────┘ │
│                                      │
│ 번역 결과 언어                       │
│ ┌─────────────────────────────────┐ │
│ │ 한국어 ✅                    ▾  │ │
│ │ ─────────────────────────────── │ │
│ │ English ✅                      │ │
│ │ 日本語 ⚠️                      │ │
│ │ 中文(简体) ⚠️                  │ │
│ │ Français ✅                     │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

**다운로드 확인 다이얼로그**:
```
┌──────────────────────────────────┐
│  ⬇️ 언어팩 다운로드               │
│                                   │
│  日本語 번역을 위해 언어팩을       │
│  다운로드해야 합니다.              │
│  (약 100MB)                       │
│                                   │
│      [취소]  [다운로드]            │
└──────────────────────────────────┘
```

**구현 위치**:
- `LanguagePackManager` 클래스 (Core/)
- `SettingsView` 수정 — 피커에 상태 아이콘 추가
- 다운로드 트리거는 `TranslationSession.Configuration`의 자동 다운로드 활용

### 3. 윈도우 크기 개선

**목적**: 설정창과 번역 팝업의 가독성을 높인다.

**현재 문제**:
- 설정창 (400×280): 항목이 많아지면 스크롤 발생
- 번역 팝업 (400×300 고정): 짧은 텍스트에는 과도하게 크고, 긴 텍스트에는 부족

**개선안**:

| 윈도우 | 현재 | 개선 |
|--------|------|------|
| 설정창 | 400×280 고정 | 450×380 기본, 콘텐츠에 맞게 조정 |
| 번역 팝업 | 400×300 고정 | 너비 320~480, 높이 콘텐츠 기반 (최소 120, 최대 400) |

**번역 팝업 크기 로직**:
```
- 로딩 상태 (인식 중/번역 중): 최소 크기 (320×120)
- 완료 상태: 텍스트 길이에 따라 유동적
  - 1~2줄: 320×160
  - 3~5줄: 400×250
  - 6줄 이상: 400×400 (스크롤)
- 에러 상태: 320×180
```

**구현 위치**:
- `SettingsView` — frame 수정
- `TranslationPopupWindow` — 동적 크기 계산 로직
- `TranslationPopupView` — idealWidth/maxHeight 조정

### 4. 원문 보기 기능

**목적**: 번역문과 원문을 나란히 비교할 수 있게 한다.

**요구사항**:
- 번역 완료 상태에서 "원문 보기" 토글 버튼 추가
- 토글 ON: 원문 (위) + 구분선 + 번역문 (아래)
- 토글 OFF: 번역문만 표시 (기본값)
- 원문도 텍스트 선택 + 복사 가능

**UI 설계**:
```
토글 OFF (기본):
┌─────────────────────────────────┐
│  등록된 프로젝트                 │
│  사용 가능한 도구 (비활성화)      │
│  사용 가능한 모드                 │
│                                  │
│  [원문 보기]  [복사]  [닫기]     │
└─────────────────────────────────┘

토글 ON:
┌─────────────────────────────────┐
│  원문                     영어   │
│  Registered Projects             │
│  Available Tools (Disabled)      │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│  번역                     한국어  │
│  등록된 프로젝트                  │
│  사용 가능한 도구 (비활성화)       │
│                                  │
│  [원문 숨기기]  [복사]  [닫기]    │
└─────────────────────────────────┘
```

**데이터 변경**:
```swift
// 기존
struct TranslationResult: Sendable {
    let text: String
    let lowConfidence: Bool
}

// 변경
struct TranslationResult: Sendable {
    let sourceText: String       // 원문 추가
    let text: String             // 번역문
    let lowConfidence: Bool
    let sourceLanguageCode: String?  // 감지된 언어
}
```

**구현 위치**:
- `TranslationCoordinator.TranslationResult` — sourceText 필드 추가
- `TranslationPopupView` — 원문 보기 토글 UI
- `TranslationPopupWindow` — 토글 시 크기 조절

### 5. 연속 번역 충돌 해결

**목적**: 새로운 번역 요청 시 이전 요청이 에러가 아닌 정상 흐름으로 처리되게 한다.

**현재 문제**:
```
[로그] 에러: translationFailed("새 번역 요청으로 취소됨")
```
- `TranslationBridge.translate()`에서 기존 continuation을 에러로 resume
- 이로 인해 coordinator가 `.failed` 상태로 전환
- 사용자에게 "번역 실패" 로 보임

**해결 방안**: 순차 취소 (이전 작업 깨끗하게 정리 → 새 작업 시작)

**변경 사항**:

1. **TranslationBridge**: 기존 continuation 취소 시 `CancellationError`를 throw
   ```swift
   // 기존: TranslationError.translationFailed("새 번역 요청으로 취소됨")
   // 변경: CancellationError()
   if let existing = continuation {
       existing.resume(throwing: CancellationError())
       continuation = nil
   }
   ```

2. **TranslationCoordinator**: `CancellationError`는 이미 `.idle`로 조용히 처리
   ```swift
   } catch is CancellationError {
       state = .idle  // 이미 구현됨 — 에러 표시 없이 조용히 종료
   }
   ```

3. **AppOrchestrator.startTranslation()**: 이미 기존 작업을 취소하고 새 작업 시작하는 로직 있음. continuation 에러 타입만 변경하면 자연스럽게 동작.

**구현 위치**:
- `TranslationBridge.translate()` — resume 에러 타입 변경 (1줄)
- 나머지는 기존 코드가 이미 처리

| 순위 | 항목 | 난이도 | 영향도 | 비고 |
|------|------|--------|--------|------|
